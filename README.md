# 수업 내용 정리
## C언어의 탄생 
### 호환성이 좋은 운영체제를 개발하기 위해 탄생
- 프로그래밍은 기계어 -> 저급언어 -> 고급언어 순으로 발전
- 최초의 운영체제는 특정 하드웨어와 1:1 맞대응 형태(저급언어 수준)
  🡪 하드웨어가 바뀌면 프로그램도 바뀌어야 함
- Ex) 두 회사의 CPU가 있을 때..(10101010 🡪 MOV?)
- C언어는 ‘컴파일’을 통해 해당 하드웨어에 맞는 기계어로 변환함
  
  기계어로 변환된 파일은 ‘목적 파일’ 이라고 읽으며,  링킹을 통해 exe 확장자(실행 파일)를 가짐

## C언어 구조
```
#include <stdio.h>
int main(void) {
    printf("Hello World!");
    return 0;
}

```
### main 함수
C 프로그램은 함수로 만들어지며, 함수는 일정 기능을 수행하는 코드 단위
프로그램의 시작을 의미, 대부분 반드시 있어야 함
함수의 범위는 중괄호로 구분
return 0; 명령어는 프로그램이 성공적으로 끝났다는 신호를 OS에 전달하고, 제어 흐름을 명확하게 종료하기 위함

### 세미콜론
각 문장의 끝에는 종료 지점이라는 표시로 세미콜론을 반드시 적어야 함.

### printf() 함수
문자열을 출력하는 함수

## 주석
- 주석
  - 소스 코드를 설명하는 내용 또는 남겨두고 싶은 메모 작성
  - 주석 처리된 문장은 프로그램이 실행될 때 영향 X
  - 여러 명이 함께 코드를 작성하는 경우 필수 작업
- 사용 방법
  - // 🡪 //부터 해당 줄의 끝까지 주석 처리
  - /* ~ */ 🡪 /*과 */ 사이의 모든 내용을 주석 처리
- 주석 단축키
  - 주석 처리하고 싶은 영역 지정 후 Ctrl+K, C(주석처리) Ctrl + K, U(주석 해제)
## 제어문자
- 제어 문자란?
  - 문자는 아니지만, 출력 방식에 영향을 주는 문자
  - 일반 문자와 구분하기 위해 백슬래시(\, 키보드에서 ￦)와 함께 사용
